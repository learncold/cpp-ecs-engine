# SafeCrowd 대피 시뮬레이션 결과 화면 지표 제안

이 문서는 SafeCrowd를 건물/행사장 군중 대피 시뮬레이션 결과를 보여주는 데스크톱 의사결정 지원 도구로 처음 설계한다고 가정하고, 결과 화면에 유용한 위험 유형, 정량 지표, 시각화 방식, 우선순위를 정리한 조사 메모이다.

저장소 내부 설계 문서, 코드, UML, Sprint 계획, Product Backlog는 고려하지 않았다. 지표는 비전문가 운영자와 안전관리자가 이해하고 행동으로 옮길 수 있는 형태를 우선했다.

## 근거 요약

Pathfinder류 도구는 밀도, 보행속도, 잔여 대피시간, 사용량, 이동 경로를 히트맵/컨투어로 보여주며, 측정 구역의 밀도와 속도 계산에도 별도 방법을 둔다. MassMotion도 혼잡 지점, 대기시간, 공간 성능 지표 분석을 강조한다. 연구 및 안전 가이드는 대피 완료시간, 구역별 대피시간, 통로 밀도, 이동거리, 불편도, 유량, 병목 유출률, ASET/RSET 안전여유를 중요 지표로 본다.

## 1. 결과 화면 핵심 지표 10개

| 우선 | 핵심 지표 | 운영자용 의미 | 권장 표시 | 1차 의사결정 |
|---:|---|---|---|---|
| 1 | 총 대피 완료시간, RSET | 마지막 사람이 안전구역에 도착하는 시간 | 큰 숫자 + 목표 대비 색상 | 계획 시간 초과 여부 판단 |
| 2 | 안전여유, ASET - RSET | 위험 조건 도달 전 얼마나 여유가 있는지 | 초/분 단위 배지 | 화재·연기 시나리오의 허용성 판단 |
| 3 | 시간별 잔류 인원 | 몇 분 뒤 몇 명이 아직 남는지 | 라인 그래프 | 안내 방송·스태프 투입 타이밍 결정 |
| 4 | 최대 군중 밀도 | 가장 위험하게 몰린 지점의 인원/m² | 지도 히트맵 + 최대값 | 압착·넘어짐 위험 구역 식별 |
| 5 | 고밀도 지속시간 | 위험 밀도가 몇 초/분 지속되는지 | 구역별 표 + 지도 | 일시적 피크와 지속 위험 구분 |
| 6 | 병목 지점 대기시간 | 문, 계단, 게이트 앞에서 얼마나 기다리는지 | 병목 랭킹 표 | 출구 개방, 유도선 변경, 스태프 배치 |
| 7 | 출구/경로 이용률 | 특정 출구에 사람이 과도하게 몰리는지 | 출구별 막대그래프 | 일부 출구로 유도 분산 |
| 8 | 통과 유량 | 출구·계단이 분당 몇 명을 처리하는지 | 시간 그래프 | 물리적 처리능력 부족 판단 |
| 9 | 저속·정체 인원 | 거의 움직이지 못하는 사람이 얼마나 되는지 | 지도 + 잔류 인원 그래프 | 정체 해소 조치 필요 여부 |
| 10 | 취약자/지원 필요자 대피시간 | 이동약자, 보조 필요자의 마지막 도착 시간 | 그룹별 완료시간 표 | 지원 인력·피난구역 계획 검토 |

## 2. 위험 유형별 보조 지표

| 위험 유형 | 보조 지표 | 비전문가용 해석 |
|---|---|---|
| 압착·과밀 | 평균/최대 밀도, 4명/m² 이상 구역, 6명/m² 이상 구역, 고밀도 지속시간 | 사람이 스스로 방향을 바꾸기 어려운 구역이 어디인지 |
| 병목 | 대기열 길이, 평균 대기시간, 최대 대기시간, 병목 유출률 | 문/계단/게이트가 사람을 처리하지 못하는지 |
| 경로 불균형 | 출구별 이용률, 미사용 출구 수, 경로별 평균 소요시간 | 열려 있는 출구가 있어도 사람들이 쓰지 않는지 |
| 정체·역류 | 저속 인원, 정지 시간, 역방향 흐름, 교차흐름 지점 | 사람 흐름이 서로 막고 있는지 |
| 계단·수직 이동 | 층별 잔류 인원, 계단 밀도, 계단 유량, 층별 완료시간 | 어느 층이 늦게 빠지는지 |
| 취약자 위험 | 그룹별 대피 완료시간, 보조 대기시간, 지원자-피지원자 거리 | 도움이 필요한 사람이 뒤처지는지 |
| 위험 노출 | 위험구역 체류 인원, 노출 시간, 안전구역 도달률 | 연기·화재·침수 등 위험과 사람이 겹치는지 |
| 운영 실패 | 닫힌 출구 영향, 안내 지연 영향, 스태프 배치별 개선량 | 운영상 어떤 조치가 가장 효과적인지 |
| 행사장 특화 | 전면 무대 밀도, 펜스 앞 압축, 매점/화장실 주변 정체 | 대피 외 평상시 혼잡 위험까지 보이는지 |

## 3. 추천 시각화 방식

| 시각화 | 보여줄 지표 | 추천 이유 |
|---|---|---|
| 평면 지도 히트맵 | 밀도, 저속, 대기시간, 위험 노출 | 운영자가 가장 빨리 위험 위치를 이해함 |
| 시간 슬라이더 애니메이션 | 인원 이동, 정체 발생·해소 | 언제 문제가 생기는지 파악 가능 |
| 병목 랭킹 표 | 대기시간, 대기열, 유량 | 조치 우선순위 결정에 적합 |
| 출구별 막대그래프 | 이용률, 처리 인원, 완료시간 | 출구 분산 실패를 즉시 확인 |
| 잔류 인원 라인 그래프 | 시간별 남은 인원 | 전체 대피 진행률을 쉽게 설명 |
| 구역별 완료시간 지도 | 층/구역별 마지막 도착시간 | 늦게 비워지는 구역 식별 |
| 시나리오 비교 표 | 기본안 vs 출구 추가 vs 안내 변경 | 의사결정 회의용으로 유용 |
| 경로 흐름 화살표 | 주요 이동 방향, 역류, 교차흐름 | 복잡한 동선 문제를 직관화 |

## 4. 문장형 해석 예시

| 상황 | 화면에 표시할 해석 문장 |
|---|---|
| 대피시간 초과 | 전체 대피 완료시간이 목표 8분보다 1분 40초 깁니다. 2층 동측 계단의 대기시간이 주요 원인입니다. |
| 과밀 지속 | 무대 전면 좌측 구역에서 5명/m² 이상 밀도가 2분 10초 지속됩니다. 이 구역은 입장 제한 또는 분산 유도가 필요합니다. |
| 출구 불균형 | 서측 출구는 처리 여유가 있지만 전체 인원의 12%만 사용합니다. 안내 표지 또는 스태프 유도로 서측 출구 이용을 늘릴 수 있습니다. |
| 병목 | B 게이트 앞 평균 대기시간이 3분 20초로 가장 높습니다. 게이트 추가 개방 시 총 대피시간 감소 효과가 큽니다. |
| 취약자 | 휠체어 이용자 그룹의 마지막 도착시간이 일반 이용자보다 4분 늦습니다. 보조 인력 또는 임시 대피공간 검토가 필요합니다. |
| 안전여유 부족 | 연기 위험 도달 예상시간보다 대피 완료가 50초 늦습니다. 현재 시나리오는 안전여유가 없습니다. |
| 정체 해소 | 안내 시작을 60초 앞당기면 1층 로비의 최대 밀도가 18% 감소합니다. |

| 고급 지표 | 초기 제외 이유 | 나중에 필요한 경우 |

| 군중 난류/파동 감지 | 비전문가 설명이 어렵고 데이터 요구 큼 | 대형 스탠딩 이벤트 |

| 실시간 센서 보정 | 하드웨어·운영 연동 필요 | 실제 행사 운영 관제 |
| route choice entropy | 해석 난이도 높음 | 경로 선택 모델 검증 |

## 6. 지표별 데이터 요구사항과 계산 난이도

| 지표 | 필요 데이터 | 계산 난이도 |
|---|---|---|
| 총 대피 완료시간 | 개인별 안전구역 도착시간 | 낮음 |
| 안전여유 | RSET, ASET 또는 목표 허용시간 | 중간 |
| 시간별 잔류 인원 | 시간별 개인 위치/상태 | 낮음 |
| 최대 밀도 | 시간별 위치, 구역 면적 또는 격자/Voronoi 밀도 | 중간 |
| 고밀도 지속시간 | 밀도 시계열, 임계값 | 중간 |
| 병목 대기시간 | 병목 위치, 개인별 통과시간, 속도 저하 기준 | 중간 |
| 출구 이용률 | 개인별 최종 출구/경로 | 낮음 |
| 통과 유량 | 출구/계단 폭, 시간별 통과 인원 | 중간 |
| 저속·정체 인원 | 개인별 속도 시계열, 정체 기준 | 낮음~중간 |
| 취약자 대피시간 | 개인 그룹 속성, 도착시간, 지원 관계 | 낮음~중간 |
| 위험 노출 | 위험구역/시간 데이터, 개인 위치 | 중간~높음 |
| 교차흐름/역류 | 이동 방향 벡터, 구역별 주 흐름 방향 | 중간~높음 |

## 결론

초기 제품에서는 정확한 물리 위험 모델보다 "어디가, 언제, 얼마나 오래, 몇 명에게 문제가 되는가"를 명확히 보여주는 것이 가장 유용하다. 특히 밀도, 지속시간, 병목 대기, 출구 불균형, 취약자 지연은 비전문가도 바로 조치로 연결할 수 있는 핵심 지표이다.
