# SafeCrowd 개요 
## SafeCrowd란?
SafeCrowd: 군중 안전 시뮬레이터

## 대상 이해당사자 (stakeholder)
- 건물 관리자 및 안전 관리자
- 시설, 행사장 운영자
- 건축·공간 설계자
- 재난 대응 담당자
- 실제 공간을 이용하는 방문객 및 시민

특히 건물이나 행사장에서는 운영자와 관리자 입장가 혼잡 구간이나 병목 구간을 사전에 예측하는 것이 중요하다. 따라서 설계나 운영 과정에서 다양한 상황에서 어떤 문제가 발생하는지, 어떤 배치와 안내 방식이 더 안전하고 효율적인지를 이해해야 한다.

## 이해당사자의 고충(pain point) 또는 니즈(needs)

### 고충
- 많은 인원의 이동시 병목 구간 파악의 어려움
- 높은 대피 훈련 비용, 반복적인 실험의 어려움
- 행사장이나 건물 구조 변경 시 안전성 영향의 빠른 검토가 어려움.
- 의사결정의 객관적인 근거 부족

### 니즈
- 다양한 시나리오를 비교하여 더 안전하고 효율적인 동선 설계
- 사용자가 자신이 관리하는 건물이나 행사장 구조를 쉽게 불러와 시뮬레이션
- 다양한 시나리오를 손쉽게 바꾸어 시뮬레이션
- 시뮬레이션 결과로 잠재적 위험 구간과 운영 대안별 차이를 파악
- 분석 결과를 바탕으로 우선 검토할 운영 대안을 시스템이 직접 추천

## 프로젝트 수행자의 의도

게임 엔진 기술을 이용하여 사회 문제 해결에 활용하려는 시도. 특히 ECS 구조는 많은 개체를 효율적으로 관리하는 데 강점이 있어, 군중 모델링 및 시뮬레이션에 적합하다고 판단.

## 주요 구현 내용
- ECS 구조를 적용한 엔진 코어 설계 및 구현
- 다수 에이전트의 이동 처리 및 성능 최적화
- 군중 행동 모델(분리, 정렬, 회피, 목표 지향 이동 등) 적용
- 병목, 정체, 고밀도 체류, 근접 노출, 운영 갈등, 탈출 지연, 미대피/제한시간 초과 같은 위험 징후를 시뮬레이션으로 비교 가능하게 구현
- 건물·행사장 구조를 반영한 맵 또는 공간 모델링
- 사용자가 건물 구조 데이터를 불러오거나 직접 설정할 수 있는 입력 기능 설계
- 다양한 시나리오와 운영 대안을 UI에서 생성·수정·저장할 수 있는 인터페이스 구현
- 출구 폐쇄, 인원 밀집, 특정 구역 통제, 장애물 추가 등 상황 변화 시뮬레이션 기능 구현
- 시뮬레이션 결과를 기준 시나리오 대비 비교 가능하게 시각화하고, 의사결정에 필요한 핵심 위험 지표를 제공하는 기능 구현
- 핵심 위험 구간과 지표를 근거로 출구 추가 개방, 순차 퇴장, 통제 구역 조정 같은 운영 대안을 추천하는 기능 구현

## 기대 결과물 및 기능

- ECS 기반 군중 시뮬레이션 실행 프로그램(프로토타입)
- 사용자가 UI만으로 건물 또는 행사장 환경을 반영한 다수의 시나리오를 직접 생성·변경·실행할 수 있는 기능
- 다수의 대피 시나리오 시뮬레이션 수행
- 시나리오별 결과와 운영 대안을 비교·분석할 수 있는 사용자 인터페이스
- 시나리오별 결과 요약, 위험 히트맵, 핵심 위험 지표 시각화 기능
- 분석 결과를 근거로 운영 대안 후보를 우선순위와 함께 추천하고, 이를 바로 대안 시나리오로 생성할 수 있는 기능

## 정량적·정성적 달성 지표
- 많은 수의 에이전트가 동작하는 다수의 시나리오를 병렬적으로 시뮬레이션할 수 있는 성능 달성 
- 시뮬레이션 결과를 바탕으로 상위 운영 대안 후보를 3개 이상 추천할 수 있는 기능 확보
- 사용자 테스트에서 건물 구조 반영과 시나리오 설정이 직관적이라는 정성적 평가 확보
- 사용자 테스트에서 위험 구간 파악과 운영 대안 비교·추천에 도움이 된다는 정성적 평가 확보
