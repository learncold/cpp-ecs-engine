## 중요! 핵심 논문 1번

### 1. 상용 시뮬레이션을 안 쓰고 SafeCrowd를 쓸 이유?
논문 본문에서 "응답자의 44% 이상은 한 가지 모델만 쓰지 않고 여러 모델을 사용한다고 답했다." -> 그 이유 Figure 8의 1,2위 답변 : 프로젝트 마다 필요한 기능이 다르다 59%, 프로젝트마다 필요한 보행자/대피 모델이 다르다 44%
### => 따라서 상용 시뮬레이션이 있다고 해서 그것만 쓰는게 아니다.

### 2. 그럼 그 중 SafeCrowd가 파고들 곳은?
사용 목적 -> Figure 6-(b)  법규/기준 적합성 검토: 63%, 신규 구조물 설계 지원: 60%, 기존 구조물 분석/개선: 48%, 연구: 25% 
### => SafeCrowd는 기존 구조물 분석/개선 니즈를 타겟하겠다.

### 3-1. 모델(이 맥락에선 소프트웨어) 선택 기준은 무엇인가?
Figure 4의 1. 검증과 타당화 2. 출력 데이터 3. 문서화 4. 지속적 개발 [각 뜻 분석 필요합니다.]
### 3-2. 만약 쓰다 다른 것으로 바꾸게 되면 선택 기준은 무엇인가?
Figure 10의 상위인 Usability and management of the input and outputs 31% = 모델의 입력을 넣고 수정하고, 결과 출력을 확인·관리·활용하는 과정의 사용 편의성 -> 즉, 편의성 및 분석 용이성
### 3-3. 사용법을 배우는 방법은?  
Figure 9의 응답자들이 사용법을 배우는 방법 대부분(72%) 독학, 코스 수강 비율 낮음 -> 사용하기 쉬울 수록 유리함을 보여줌.
### => SafeCrowd입력, 시나리오 설정 및 분석이 쉽도록 만들어서 사람들이 선택하도록 하겠다.
