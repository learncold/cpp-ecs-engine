# 아이디어 발산에 포스트잇 추가할 내용들
1. 사용자와 현장 상황
- 행사 종료 직후 짧은 시간에 출구로 인파가 몰리는 상황
- 출구 일부가 폐쇄됐을 때 병목이 어떻게 바뀌는지 보고 싶다
- 설계안 A와 B 중 어느 쪽이 더 안전한지 빠르게 비교하고 싶다
- 실제 훈련 없이 대피 계획을 사전 검토하고 싶다
- 안전관리자는 모델링 전문가가 아니어도 써야 한다
- 사용자에게 중요한 것은 “정확한 내부 알고리즘”보다 “어디가 왜 위험한지”이다
2. 입력과 모델 생성
- CAD/BIM을 불러오면 보행 가능 영역을 자동 추출하면 좋겠다
- 출구, 계단, 장애물 후보를 자동으로 인식하면 좋겠다
- import 결과가 바로 usable model이 되지 않으니 수정 가이드가 필요하다
- 누락된 출구나 막힌 통로를 자동 경고하면 좋겠다
- 구조 데이터가 없어도 간단한 편집기로 직접 그릴 수 있어야 한다
- 자동 생성 후 사람이 마지막으로 검수하는 흐름이 현실적이다
3. 시나리오 정의
- 화재, 행사 퇴장, 출구 폐쇄 같은 템플릿이 있으면 좋겠다
- 기준 시나리오를 복제해서 한 변수만 바꾸게 해야 한다
- 인원 수, 인원 분포, 이동 목적지 프리셋이 필요하다
- 시간에 따라 사건이 바뀌는 이벤트 타임라인이 있으면 좋겠다
- “어떤 변수를 바꾸면 되는지”를 UI가 먼저 제안해주면 좋겠다
- 비전문가는 자유 입력보다 가이드형 입력이 더 쉽다
4. 실행과 비교
- 여러 시나리오를 한 번에 돌리고 비교하고 싶다
- 기준안 대비 얼마나 나빠졌고 좋아졌는지 바로 보여줘야 한다
- 동일 조건 비교를 위해 랜덤 시드 고정 기능이 필요하다
- 결과를 replay 형태로 다시 볼 수 있으면 설명하기 쉽다
- 위험 구간이 나타난 시점을 타임라인으로 보여주면 좋겠다
- 배치 실행보다 “비교 가능한 실험 구조”가 더 중요하다
5. 결과 해석과 의사결정
- 병목 위치를 자동으로 하이라이트하면 좋겠다
- 최대 밀도, 압력 집중, 대피시간, 미대피 인원을 핵심 지표로 요약하자
- 단순 수치보다 “왜 이 지점이 위험한지” 설명이 필요하다
- 대안별 장단점을 한 화면에서 비교해야 한다
- 보고서용 한 장 요약이 있으면 발표와 실무 둘 다 좋다
- 사용자에게는 raw simulation보다 actionable insight가 중요하다
6. 차별화와 발표 메시지
- SafeCrowd는 범용 시뮬레이터가 아니라 안전 의사결정 지원 도구다
- 핵심은 군중을 움직이는 것이 아니라 위험을 설명 가능하게 만드는 것이다
- 비전문가도 첫 시나리오를 짧은 시간 안에 만들 수 있어야 한다
- 결과는 분석가만 이해하는 그래프가 아니라 운영자가 판단할 수 있는 형태여야 한다
- 기존 제품이 무거운 워크플로우를 요구한다는 점이 기회다
- 우리 데모는 “입력 단순화 → 시나리오 단순화 → 해석 단순화”로 보여주는 게 좋다

# 아이디어 수렴 결과
| 아이디어 그룹 | 수렴된 아이디어 목록 |
|---|---|
| 핵심 사용자와 사용 맥락 | 행사장 운영자, 안전관리자, 설계자, 재난 대응 담당자를 핵심 사용자로 둔다.<br>공통 니즈는 위험 구간 식별, 대안 비교, 의사결정 근거 확보이다.<br>대표 상황은 행사 퇴장, 비상 대피, 설계안 비교로 수렴한다. |
| 입력 모델링 간소화 | CAD/BIM import 후 보행공간, 출구, 장애물을 자동 추출한다.<br>자동 생성 결과를 사람이 수정할 수 있는 검수 단계가 필요하다.<br>구조 데이터가 없을 때를 위한 수동 편집 fallback도 포함한다. |
| 시나리오 작성 간소화 | 템플릿 기반 시나리오 생성 기능을 둔다.<br>기준 시나리오 복제 후 변수만 바꾸는 방식으로 비교 실험을 쉽게 만든다.<br>인원 수, 분포, 출구 상태, 통제 구역, 이벤트를 가이드형 UI로 설정하게 한다. |
| 실행 및 실험 비교 | 여러 시나리오를 동일 조건에서 실행하고 baseline 대비 비교한다.<br>랜덤 시드 고정, replay, 타임라인 기반 확인 기능을 우선 고려한다.<br>핵심은 “많이 돌리는 것”보다 “공정하게 비교하는 것”이다. |
| 결과 해석과 시각화 | 병목, 밀도, 압력 집중, 대피시간, 미대피 인원을 핵심 지표로 삼는다.<br>위험 히트맵과 구간별 원인 설명을 함께 제공한다.<br>대안별 결과를 한 화면 또는 한 장 요약으로 비교 가능하게 한다. |
| 의사결정 지원 방식 | 시스템은 단순 시뮬레이션 결과가 아니라 운영 대안 선택 근거를 제공해야 한다.<br>“어느 대안이 더 안전한가”와 “왜 그런가”를 함께 보여준다.<br>보고서나 발표에 바로 쓸 수 있는 요약 출력이 중요하다. |
| SafeCrowd 차별화 포인트 | SafeCrowd는 범용 전문가용 시뮬레이터가 아니라 비전문가 친화적 의사결정 지원 도구로 정의한다.<br>차별화의 핵심은 위험 중심 지표, 설명 가능성, 쉬운 시나리오 수정이다.<br>발표 메시지는 “입력, 설정, 해석의 복잡성을 줄인다”로 수렴한다. |
| MVP 우선순위 | 1차 구현은 단일 공간 입력, 시나리오 템플릿, 시나리오 비교, 핵심 위험 지표 시각화에 집중한다.<br>과도한 확장 기능보다 발표 메시지와 직접 연결되는 기능을 우선한다.<br>즉, `usable model 생성`, `쉬운 시나리오 정의`, `결과 비교`가 MVP의 중심이다. |
